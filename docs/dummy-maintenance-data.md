# 더미 유지보수 벡터 데이터

## 목적

LEVEL 5 작업으로 ChromaDB `maintenance_logs` 컬렉션에 유지보수 이력 더미 데이터
101건을 저장했다. 이 데이터는 분석 payload의 `sourceId`와 일치하면
`/llm/analyze/preview`, `/llm/analyze`에서 `past_history`로 조회되어 LLM 입력에
붙을 수 있도록 만든 테스트용 이력이다.

## 생성 기준

- 기준 배관도: `prompt/prompt.md`의 LEVEL 5 배관도 JSON
- 대상 시설: 배관도 `nodes[]` 중 `type`이 `pipeSegment`, `connector`,
  `elbowConnector`인 항목
- 사용 시설 수: 76개
- 생성 건수: 101건
- 저장 컬렉션: `maintenance_logs`
- 저장 경로: `.env`의 `CHROMA_PERSIST_DIR`, 로컬 기본값은 `./chroma_data`
- 임베딩 모델: `text-embedding-3-small`

각 더미 이력의 `event.target_id`는 배관도 node의 `id`를 사용했다. 따라서 같은 ID가
분석 입력의 `link[].id` 또는 `swmmIssues[].primaryElement.id`로 들어오면 기존
로직에서 과거 이력으로 조회된다.

## 포함 내용

각 payload는 `maintenance_store.py`의 `format_maintenance_case()`를 거쳐 아래
형태의 텍스트로 저장됐다.

- 이벤트 ID: `DUMMY-MAINT-001`부터 `DUMMY-MAINT-101`
- 시설 ID: 배관도 node `id`
- 시설 유형: `pipeSegment`는 `link`, connector 계열은 `editor_object`
- 위험 유형: `REVERSE_FLOW`, `PREDICTED_FULL_PIPE`,
  `PREDICTED_CAPACITY_EXCEEDED`, `BLOCKAGE`, `HIGH_DEPTH` 순환
- 위험 등급: `CRITICAL`, `HIGH`, `MEDIUM` 조합
- 주요 지표: 유량, 흐름 방향, 만관율, 용량 비율, 막힘 비율, 수위 비율, 침수 유량
- 조치 내용: 현장 점검, 토사/낙엽 제거, 임시 배수 확보, 배수펌프 대기 등
- 결과: 유량 안정, 수위 상승 둔화, 만관율 감소, 반복 알림 중지 등
- 재발 시 참고사항: 강우 예보 시 우선 점검 대상, 반복 알림 시 확인 항목 등

## Metadata

ChromaDB metadata에는 다음 필드를 넣었다.

| 필드 | 설명 |
| --- | --- |
| `sourceId` | 조회 매칭에 사용하는 시설 ID |
| `loggedAt` | 더미 데이터 저장 시각 |
| `eventId` | `DUMMY-MAINT-001` 형식의 이벤트 ID |
| `hazardType` | 위험 유형 |
| `hazardLevel` | 위험 등급 |
| `actionStatus` | `RESOLVED` 또는 `MONITORING` |
| `resultStatus` | `RESOLVED` 또는 `MONITORING` |
| `priorityBand` | `P1`, `P2`, `P3` |
| `priorityScore` | 문자열로 저장한 우선순위 점수 |
| `dummyData` | 더미 데이터 식별값, `true` |

## 검증 결과

삽입 후 로컬 ChromaDB 기준으로 다음을 확인했다.

```text
maintenance_logs count = 101
sourceId pipe_free_1781761486031 조회 결과 = 2건
sourceId REL_061_CONDUIT 조회 결과 = 1건
```

`maintenance_store.get_maintenance_history_by_source_ids()`로
`pipe_free_1781761486031`을 조회했을 때 2건이 반환됐고, 반환된
`action_details`에 `조치:`, `결과:`, `재발 시 참고사항:` 섹션이 포함되어 있음을
확인했다.

## 주의사항

이 데이터는 실제 운영 이력이 아니라 로컬 테스트용 더미 데이터다. 운영 환경에
반영하기 전에 `dummyData=true` metadata 또는 `DUMMY-MAINT-*` 이벤트 ID를 기준으로
구분해야 한다.
